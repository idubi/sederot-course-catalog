# קטלוג התוכניות והקורסים — מרכז המצוינות שדרות

אתר קטלוג סטטי בעברית ובכיווניות RTL, המיועד לבחירת תוכנית, שכבה וקבוצת קהל לפני צפייה באשכול הקורסים המתאים.

## מצב הפרויקט

TASK-003 מקים את מעטפת Astro הסטטית. תוכן הקטלוג, מסלול הבחירה, עמודי התוכנית והרישום יתווספו במשימות הייעודיות לפי [`artifacts/DEVELOPMENT_TASKS.md`](artifacts/DEVELOPMENT_TASKS.md).

הקובץ `index.html` בשורש הוא שאלון האפיון העצמאי הקודם ונשמר לעיון. אתר הקטלוג החדש נבנה מתוך `src/` אל `dist/`.

## דרישות פיתוח

- Node.js 22 ומעלה
- npm 10 ומעלה

הגרסאות הנתמכות מוגדרות גם בשדה `engines` של `package.json`, ותלויות הפיתוח נעולות ב-`package-lock.json`.

## הפעלה

```bash
npm install
npm run dev
```

שרת הפיתוח נקשר ל-`127.0.0.1` בלבד כברירת מחדל.

### עורך תוכן מקומי

```bash
npm run editor
```

עורך התוכן הוא יישום React נפרד תחת `tools/content-editor/`. גם הממשק וגם API
הפיתוח שלו זמינים ב-`http://127.0.0.1:4333`, נקשרים במפורש לכתובת ה-loopback
ואינם מיועדים לגישה מהרשת. זמינים
`GET /api/health`, טעינת תוכן מאושר או טיוטה, אימות, שמירת טיוטה וייצוא JSON
מאושר. שינויים בטקסט נשמרים אוטומטית ב-Local Storage של הדפדפן; כתיבה לקבצים
מתרחשת רק בפעולה מפורשת.

ניתן לטעון JSON גם באמצעות בוחר קבצים מהמחשב או מכונן מקומי/מסונכרן, או מכתובת
HTTPS שמאפשרת CORS. הקובץ מפוענח כ-JSON לפני שהוא מחליף את תוכן העורך.
ניתן גם לטעון ישירות מסמך Word מסוג DOCX, או מסמך Blueprint בפורמט Markdown או
טקסט. קובץ המקור נקרא בזיכרון ואינו משתנה. העורך מחלץ כותרות, פסקאות, רשימות
וטבלאות, ממיר אותן לטיוטת ה-Schema הקרובה ביותר ומציג אבחונים מובנים לכל שדה
חסר או ערך לא ודאי. לאחר ההמרה ממשיכים לערוך את אותו JSON בממשק.
פעולת "איפוס העורך" דורשת אישור ומנקה רק את התוכן והשמירה האוטומטית בדפדפן;
היא אינה מוחקת טיוטה או תוכן מאושר מהכונן.

העורך אינו נתיב Astro ואינו נכלל ב-`dist/`. פלט בנייה מקומי שלו, אם נוצר לצורך
בדיקה, נשמר תחת `tools/content-editor/.local-dist/` ומוחרג מ-Git.

שמירת טיוטה כותבת אטומית רק אל `content/draft/editor/catalog.json`. ייצוא מאושר
כותב אטומית רק אל `content/approved/catalog.json`, ורק לאחר מעבר Schema מלא.
שגיאות חוסמות ייצוא; אזהרות דורשות סימון אישור מפורש. הפלט דטרמיניסטי ומסתיים
בשורה חדשה.

לצורך סביבת פיתוח מבודדת ניתן לייצא במפורש מהעורך קובץ Schema תקין אל
`contents/<folder-name>/bootstrap.json`. הייצוא אינו משנה את הקטלוג המאושר.
לאחר מכן מפעילים את האתר מול הקובץ הזה כך:

```bash
npm run dev -- --data=./contents/<folder-name>
```

הנתיב חייב להיות תיקייה ישירה בתוך `contents/` ובה קובץ בשם המדויק
`bootstrap.json`. מצב `--data` מיועד לפיתוח בלבד; Build של production דוחה אותו
וממשיך לקבל רק את `content/approved/catalog.json`.

תיאורי קורס מסוננים בעורך ובייצוא לפי allowlist מצומצם של `p`, `br`, `strong`,
`em`, `ul`, `ol`, `li` ו-`a`; קישורים מותרים ב-HTTPS בלבד ומקבלים הגנות
`noopener noreferrer`. ה-Build דוחה HTML שלא עבר סינון, נתיבים שאינם תחת
`/content/images/`, וקובצי תמונה מאושרים שמוזכרים ב-JSON אך חסרים בכונן.

הממשק המובנה מאפשר לערוך תוכניות, קבוצות קהל, תוכן קורס משותף ושיוכי קורס
לקבוצות. שינוי מזהה תוכנית, קבוצה או קורס מעדכן את ההפניות התלויות. לכל שיוך
נשמרים קבוצה, מחצית וסדר עצמאיים; ניתן לערוך נתוני תמונה כללית לקורס או תמונה
חלופית לשיוך. ניתן להעלות JPEG, PNG או WebP עד 5 MiB; הקובץ נשמר בשם יציב תחת
`public/content/images/`, והנתיב מתעדכן ב-JSON רק לאחר הצלחת ההעלאה. תצוגה
מקדימה מציגה תמונת שיוך לפני תמונת ברירת המחדל של הקורס, וללא שתיהן לא מוצגת
תמונה — מצב תקין שאינו חוסם ייצוא.

ניהול הרישום בעורך נשאר ברמת התוכנית והקבוצה בלבד: ניתן לערוך יעד HTTPS,
סוג, טקסט מידע/פעולה ומצב פעיל, לבחור ברירת מחדל לתוכנית ויעד עוקף לקבוצה,
ולבדוק בתצוגה מקדימה את היעד האפקטיבי. שינוי מזהה יעד מעדכן את הפניות התוכנית
והקבוצה; קורסים ושיוכים אינם מקבלים יעד רישום.

## בנייה סטטית

```bash
npm run build
```

התוצר נכתב ל-`dist/`. מעטפת TASK-003 כוללת CSS מוטמע וללא JavaScript או API בזמן ריצה, ולכן ניתן לפתוח את `dist/index.html` ישירות בדפדפן לצורך בדיקת המעטפת.

מעטפת הקטלוג הציבורית טוענת בזמן Build רק את `content/approved/catalog.json`
ומפיקה View Models נפרדים לתוכניות, קבוצות קהל ושיוכי קורסים. דף הבית מציג
סיכום תוכניות מאושר ב-RTL וללא JavaScript לקוח; מסלול הבחירה והעמודים המקושרים
מתווספים במשימות TASK-021–TASK-024.

מסלול הבחירה הציבורי מציג תוכנית, שכבה וקהל כתיבות תלויות ונגישות. ניווט לעמוד
הקבוצה מתאפשר רק כאשר שלושת הערכים מתאימים לקבוצה מאושרת; בחירת הקהל היא הצהרה
עצמית לצורכי ניווט, אינה מאומתת ואינה נשמרת.

לכל קבוצת קהל מאושרת נוצר עמוד סטטי תחת `/programs/[groupId]/`, המציג פרטי
תוכנית וקבוצה, אשכול קורסים לפי סדר התצוגה, פרטי קשר ופעולת הדפסה. התחלת רישום
קיימת רק בכותרת עמוד הקבוצה ומובילה תחילה לעמוד מידע פנימי; כרטיסי הקורסים
מציעים פרטים בלבד.

לכל שיוך קורס מאושר נוצר גם עמוד פרטים הקשרי תחת
`/programs/[groupId]/courses/[offeringId]/`. העמוד מציג את תמונת השיוך או את
תמונת ברירת המחדל של הקורס, תיאור מסונן, מנחים ומחצית, ומחזיר רק לקבוצת
התוכנית שממנה נפתח. אין בעמוד פרטי קורס פעולת רישום או יעד רישום.

התחלת הרישום בעמוד הקבוצה פותחת תמיד תחילה עמוד מידע פנימי תחת
`/programs/[groupId]/registration/`. יעד הרישום נפתר קודם מהקבוצה ולאחר מכן
מברירת המחדל של התוכנית. רק עמוד המידע מציג את קישור ה-HTTPS החיצוני המאושר;
סגירה או חזרה מובילות לאותה קבוצת תוכנית, והאתר אינו אוסף פרטי רישום או תשלום.

עמוד הקבוצה ועמוד פרטי הקורס מציגים קישורי טלפון, WhatsApp ודוא״ל נגישים מתוך
פרטי הקשר המאושרים. הודעות WhatsApp ודוא״ל מקבלות מראש הקשר לא-רגיש של התוכנית
והשכבה, ובעמוד קורס גם את שם הקורס. היישומים נפתחים ישירות במכשיר; האתר אינו
שולח הודעות ואינו שומר פרטי פנייה.

פעולת ההדפסה בעמוד קבוצה פותחת את כל קורסי הקבוצה ב-`/print/?group=...`, ופעולת
ההדפסה בעמוד פרטי קורס מוסיפה גם את מזהה השיוך ומציגה קורס אחד. התצוגה כוללת
כותרת, הקשר תוכנית, מועד, מנחים, מחצית, תיאור ותאריך הפקה. CSS ייעודי ל-A4
מסתיר ניווט, הוראות וכפתורים ומונע ככל האפשר פיצול קורס בין עמודים; ההדפסה או
השמירה כ-PDF מתבצעות בדפדפן בלבד, ללא יצירת PDF בשרת.

הממשק מותאם מ-320px ומעלה, נשאר RTL בכל המסלולים, ותומך בזרימת מקלדת עם קישור
דילוג, סדר מסמך סמנטי, תוויות מפורשות, הודעות מצב חיות ומסגרת מיקוד בולטת.
פעולות מרכזיות שומרות על יעד מגע של 44px לפחות, טקסט עברי וכתובות ארוכות
נשברים בלי גלילה אופקית, וקישורים שנפתחים בחלון חדש מוכרזים לקורא מסך. הגדרות
מערכת להפחתת תנועה ולצבעים מאולצים מקבלות תמיכה בסיסית; אין הצהרת תאימות
פורמלית לתקן נגישות.

עמודים מקוננים עתידיים וקישורים ביניהם ייבדקו דרך אירוח HTTP סטטי, מפני שניווט רב-עמודי אינו אמין בפרוטוקול `file://` בכל הדפדפנים.

## GitHub Pages ותת-נתיב

הגדרת `PUBLIC_BASE_PATH` מאפשרת לבנות תחת נתיב repository בלי לשנות את הקוד:

```bash
PUBLIC_BASE_PATH=/sderot-course-catalog/ npm run build
```

לאתר בדומיין או בתת-דומיין עצמאי אין צורך בערך נוסף; ברירת המחדל היא `/`. הפריסה עצמה תוגדר במשימות השחרור.

סביבת הפרסום הזמנית היא GitHub Pages בכתובת
`https://idubi.github.io/sederot-course-catalog/`. רק Push מאושר ל-`main` שעבר
את שער `Production gates` יכול ליצור תוצר Pages ולפרוס אותו. לאחר הפריסה רצה
בדיקת HTTPS מרוחקת למסלולי הבית, הקבוצה, הקורס, מידע הרישום וההדפסה. נוהל
הראיות והשחזור מתועד ב-`artifacts/deployment-runbook.md`.

## בדיקות איכות

```bash
npm run typecheck     # בדיקת TypeScript ו-Astro במצב strictest
npm run lint          # ESLint לקובצי TypeScript/Astro + אימות Prettier
npm run test          # בדיקות Vitest דטרמיניסטיות (גם לפני הוספת קובצי test)
npm run test:e2e      # Build + זרימות Chromium מול Preview ועורך מקומי
npm run test:e2e:install # התקנה חד-פעמית של Chromium עבור Playwright
npm run production:verify # בדיקת קישורים וגבולות התוצר שכבר נבנה
npm run production:gate # שער פרסום מלא, כולל Build, תוצר ו-E2E
npm run deployment:smoke -- https://host/base/ # בדיקת עשן מרוחקת לפריסה
npm run content:validate # אימות content/approved/catalog.json בלבד
npm run content:normalize-source # יצירת טיוטת תוכניות/קבוצות/קורסים/מופעים מקומית
npm run content:import-draft # יצירת טיוטה ואבחונים מובנים מקומיים
npm run content:migrate-baseline # המרת baseline תשפ״ז לטיוטת Schema ואבחונים מקומיים
npm run format:check  # אימות עיצוב קוד ללא שינוי קבצים
npm run format        # תיקון עיצוב קוד מקומי
npm run check         # שער מקומי מלא: typecheck, lint, tests ו-build
```

`npm run build` מריץ בדיקת טיפוסים לפני יצירת `dist/`, כדי שבנייה לא תצליח כאשר קיימות שגיאות TypeScript. ESLint בודק קובצי JavaScript, TypeScript ו-Astro, ו-Prettier אוכף עיצוב דטרמיניסטי. קובצי build, תלויות, מסמכי מקור וה-HTML הישן מוחרגים במפורש.

`npm run production:gate` הוא תנאי הפרסום הקנוני. הוא מאמת את ה-JSON המאושר,
מריץ בדיקות Astro, יחידה ורכיבים, בונה את האתר, בודק שכל קישור פנימי וקישור עוגן
נפתרים ושכל URL חיצוני משתמש ב-HTTPS או בסכימת קשר מאושרת, דוחה כלי עריכה,
טיוטות, אבחונים ומסמכי מקור מתוך `dist/`, ולבסוף מריץ את זרימות ה-E2E מול
התוצר הסטטי. אותו שער רץ ב-GitHub Actions בכל Pull Request אל `main` ובכל Push
אליו; רק לאחר הצלחה מועלה `dist/` כארטיפקט קצר-טווח למשימות הפריסה.

Vitest נטען דרך תצורת Astro ב-`vitest.config.ts`, ולכן בדיקות הרכיבים מרנדרות
קובצי `.astro` אמיתיים. הכיסוי כולל את מסלול הבחירה, קיבוץ וסדר שיוכים, הכרעת
תמונות, פתרון יעד רישום, Schema, כרטיס קורס, פעולות קשר ומסך מידע לפני רישום,
כולל בדיקות רגרסיה שמונעות פעולת או יעד רישום ברמת קורס או שיוך.

בדיקות Playwright מפעילות במקביל את `astro preview` על `127.0.0.1:4321` ואת
העורך המקומי על `127.0.0.1:4333`. הן מכסות בחירה עד חזרה מפרטי קורס, תיווך
מידע לפני יעד רישום חיצוני, תצוגת הדפסה, רוחב 320px וטעינה/אימות בעורך. תרחיש
העורך אינו מפעיל שמירת טיוטה או ייצוא מאושר ולכן אינו כותב קובצי תוכן. דוחות
וכשלי Playwright נשמרים מקומית ומוחרגים מ-Git.

## גבולות תוכן

ה-Build קורא קטלוג רק מתוך `content/approved/catalog.json` ומאמת אותו לפני Astro. הקובץ הנוכחי הוא Seed טכני שמסומן במפורש כלא מיועד לפרסום. קובצי עבודה תחת `content/draft/` ודוחות תחת `content/diagnostics/` נשארים מקומיים, אינם נשמרים ב-Git ואסור שייכללו ב-`dist/`.

Baseline התוכן שבחר הבעלים לשנת תשפ״ז נשמר ללא שינוי תחת
`content/baseline/catalog_2026-2027.json`, עם SHA-256
`cc7440cc19542e8a1aca012c1cba9ca79edbf437f26facc4138f10409755a5ca`.
הוא אינו קלט Build ואסור שייכלל ב-`dist/`. הפקודה
`npm run content:migrate-baseline` מאמתת את ה-hash ומפיקה באופן דטרמיניסטי
טיוטת Schema מקומית ב-`content/draft/baseline/catalog.json` ואבחונים ב-
`content/diagnostics/baseline-migration.json`. אפשר גם לבחור את קובץ ה-baseline
ישירות בעורך; ההמרה מתבצעת דרך API ה-loopback, וכל החסמים מוצגים לפני ייצוא.
אין קידום אוטומטי ל-`content/approved/catalog.json`.

מקור הייבוא המאושר הוא `artifacts/2027 cources details - blueprint.md`. לאחר כל תוספת או שינוי מאושר במקור, מפיקים מחדש snapshot דטרמיניסטי ושומר-מיקום באמצעות:

```bash
npm run content:read-source
```

הפלט נכתב תמיד אל `content/draft/source-reader/blueprint-document.json`. התיקייה כולה מקומית ומוחרגת מ-Git; הקובץ ניתן לבנייה מחדש בכל עת מן המקור הקנוני ואינו קלט ל-Build הציבורי.

לשנת לימודים עתידית מתחילים מעותק של `artifacts/course-blueprint-template.md`, שומרים אותו כקובץ מקור שנתי נפרד ומקבלים אישור מפורש לנתיב ול-hash החדש לפני שינוי חוזה הקורא. אין למלא נתוני אמת בתוך קובץ התבנית ואין לדרוס מקור שנתי קודם.
